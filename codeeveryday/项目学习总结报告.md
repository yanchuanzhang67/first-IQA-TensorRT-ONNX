## a. 项目背景与目标

### 项目背景

IQAstudy 是一个**图像质量评估（Image Quality Assessment, IQA）**的学习项目，旨在通过系统性的代码实践，掌握从深度学习基础到模型部署的完整技术栈。

图像质量评估是计算机视觉领域的重要研究方向，主要任务是通过算法自动预测人类对图像质量的感知分数。IQA 技术广泛应用于：

- **图像压缩**：评估压缩算法的质量损失
- **图像增强**：评估增强后图像的画质提升
- **图像生成**：评估 GAN、扩散模型生成图像的质量
- **视频处理**：评估视频转码、传输后的质量

### 项目目标

本项目的学习目标包括：

1. **掌握 Python 面向对象编程** - 理解类、继承、封装等概念
2. **掌握 NumPy 与 PyTorch 基础** - 理解张量操作、GPU 加速、自动求导
3. **掌握数据处理流程** - Dataset、DataLoader、数据增强
4. **掌握模型训练与评估** - 训练循环、损失函数、评价指标
5. **掌握迁移学习技术** - 使用预训练模型（ResNet、ViT）
6. **掌握模型部署** - ONNX 导出、C++ 推理

### 学习路径设计

项目采用**循序渐进**的设计思路，每天一个主题，共 18 天的学习内容：

| 阶段 | 内容 | 天数 |

## d. 核心代码与算法

### 核心算法

#### 1. 图像质量评价指标（Day 7）

IQA 任务使用两个核心指标：

**SRCC (Spearman Rank Correlation Coefficient)**
- 中文名：Spearman 等级相关系数
- 作用：衡量预测分数与真实分数的**排名一致性**
- 特点：对分数的线性偏移不敏感
- 公式：$\rho = 1 - \frac{6\sum d_i^2}{n(n^2-1)}$
- 理想值：1.0（完全正相关）

**PLCC (Pearson Linear Correlation Coefficient)**
- 中文名：Pearson 线性相关系数
- 作用：衡量预测分数与真实分数的**线性拟合度**
- 特点：对分数偏移敏感
- 公式：$r = \frac{\sum(x_i-\bar{x})(y_i-\bar{y})}{\sqrt{\sum(x_i-\bar{x})^2}\sqrt{\sum(y_i-\bar{y})^2}}$
- 理想值：1.0（完美线性关系）

```python
def compute_metrics(pred, target):
    srcc, _ = stats.spearmanr(pred, target)
    plcc, _ = stats.pearsonr(pred, target)
    return srcc, plcc
```

#### 6. C++ 图像预处理（Day 18）

```cpp
std::vector<float> Preprocess(cv::Mat& img) {
    cv::resize(img, img, cv::Size(224, 224));
    cv::cvtColor(img, img, cv::COLOR_BGR2RGB);
    
    float mean[] = {0.485, 0.456, 0.406};
    float std[] = {0.229, 0.224, 0.225};
    
    std::vector<float> tensor;
    for (int h = 0; h < 224; h++) {
        for (int w = 0; w < 224; w++) {
            for (int c = 0; c < 3; c++) {
                float pixel = (img.at<cv::Vec3b>(h, w)[c] / 255.0f - mean[c]) / std[c];
                tensor[c * 224 * 224 + h * 224 + w] = pixel;
            }
        }
    }
    return tensor;
}
```

## e. 遇到的挑战与解决方案

### 挑战 1：NumPy 与 PyTorch 张量维度转换

**问题**：图像数据从 HWC 格式转换为 CHW 格式时容易混淆。

**解决方案**：
- HWC（高度-宽度-通道）：图片原始格式
- CHW（通道-高度-宽度）：深度学习框架格式
- 使用 `transpose(2, 0, 1)` 进行转换

### 挑战 2：GPU 与 CPU 设备兼容

**问题**：在 GPU 上训练的模型无法直接在 CPU 上加载。

**解决方案**：
```python
device = torch.device("cuda" if torch.cuda.is_available() else "cpu")
model.load_state_dict(torch.load("model.pth", map_location=device))
```

### 挑战 3：训练时梯度累加

**问题**：如果不清零梯度，梯度会累加导致训练错误。

**解决方案**：
```python
optimizer.zero_grad()  # 每个 batch 前清零梯度
```

### 挑战 4：模型.eval() 与训练模式混淆

**问题**：推理时忘记切换到 eval 模式，导致 BatchNorm 和 Dropout 行为异常。

**解决方案**：
```python
model.eval()  # 推理前切换
with torch.no_grad():
    output = model(input)
```

### 挑战 5：ONNX 动态 batch 支持

**问题**：导出的 ONNX 模型默认只支持固定 batch size。

**解决方案**：设置 `dynamic_axes` 参数：
```python
dynamic_axes={'input': {0: 'batch_size'}, 'output': {0: 'batch_size'}}
```

### 挑战 6：C++ 图像格式转换

**问题**：OpenCV 读取的图像是 HWC BGR 格式，需要转换为 CHW RGB 格式。

**解决方案**：手动实现像素重排（Day 18 的 Preprocess 函数）

### 挑战 7：学习率设置不当

**问题**：学习率过大导致 loss 不收敛，学习率过小导致训练太慢。

**解决方案**：使用学习率调度器（Day 14），初始设高学习率，后期衰减。

### 常见错误排查

| 错误 | 原因 | 解决方案 |
|------|------|----------|
| CUDA out of memory | batch size 过大 | 减小 batch size |
| RuntimeError: expected dtype=float32 | 数据类型不匹配 | 使用 `dtype=torch.float32` |
| 模型输出 NaN | 学习率过大或数据未归一化 | 调小学习率，检查 Normalize |
| ONNX export failed | 模型有动态操作 | 避免动态操作或设置 dynamic_axes |
